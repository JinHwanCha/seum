📘 SEUM 웹앱 기획 & 개발 통합 문서 (v1.0 / MVP)
1. 서비스 개요

서비스명: SEUM (세움)
목적:
교회 내 공동체(특히 청년부)의 기도제목 공유 및 커뮤니케이션 통합 플랫폼

핵심 가치

분산된 기도제목 → 한눈에 보기
공동체 구조 기반 권한 시스템
교회별 독립 운영 (멀티 테넌시 구조)
2. URL 구조 (멀티 테넌시)
seum.kr/{church}/{department}

예시:

seum.kr/naesoodong/fishermen (청년부)
seum.kr/naesoodong/witness (대학부)
✅ 로그인 리디렉션
로그인 성공 시:
→ 해당 유저의 church + department 자동 이동
3. 사용자 구조 (권한 시스템 핵심)
3.1 Role Hierarchy (권한 레벨)
[최상위]
사역자
  └ 목사
  └ 강도사
  └ 전도사
  └ 간사

국장단 (행정)

마을장 (대그룹장)

목자 (소그룹 리더)

목원 (일반 성도)
3.2 권한 핵심 규칙
역할	권한
사역자	전체 권한 (유저/조직/게시글/기도제목)
국장단	공지 작성, 일부 관리
마을장	마을 전체 관리
목자	소그룹 관리
목원	기본 사용자
3.3 직급 변경 규칙 (중요)
상위 직급만 하위 직급 수정 가능
예:
목사 → 전도사 변경 가능
전도사 → 목사 변경 ❌

👉 RBAC + Hierarchy 구조 필수

6. 기도제목 시스템 (핵심 기능)
6.1 기준
주 단위 관리 (일요일 기준)
월별 주차별 기도제목 확인 가능
예:
2026-03-22~2026-03-28 기도제목 기입 시 3월 4주차(2026-03-22) 기도제목
2026-03-29~2026-04-04 기도제목 기입 시 3월 5주차(2026-03-29) 기도제목

6.2 작성 규칙
역할	권한
본인	작성/수정
목자	소그룹 전체 수정 가능
목원	소그룹 전체 작성 가능
6.3 조회 권한
역할	조회 범위
목원	소그룹
목자	마을 전체
마을장	마을 전체
국장단	국장단 + 사역자
사역자	전체
목사	전체 공개

👉 이건 권한 필터링 쿼리 설계 핵심 포인트

7. 게시판 시스템
7.1 공통 기능
댓글 / 대댓글
이모지 반응
작성자 수정/삭제
관리자 삭제 가능
7.2 게시판 종류
1) 공지 게시판
작성: 사역자, 국장단
수정/삭제: 작성자 + 사역자
2) 나눔 게시판
작성: 전체
삭제: 사역자, 마을장
카테고리:
대예배 / 새벽기도 / 말씀묵상 등
카테고리 관리: Admin
3) 모임 게시판
자유 형식
"어떤 모임인가요?" 입력 기반
카테고리 강제 ❌
4) 중보기도 게시판
작성: 전체
삭제: 사역자, 마을장
